requireJS Optimization Config Options

Ref: https://github.com/jrburke/r.js/blob/master/build/example.build.js

appDir
The top level directory that contains the entire app
notes

Usage

  • Example - using appDir appDir: "../app/"

Notes

  • If this option is used then it assumed the scripts file are in a subdirectory under this path.
  • If this option is specified, then all the files from the app directory will be copied to the directory specidified using dir config option.
close
baseUrl
Specifies the base path for all script modules
notes

Usage

  • Example - using baseUrl along with appDir config option appDir: "../app", baseUrl: "public/scripts"
  • Example - using baseUrl without specifying appDir config option baseUrl: "../app/public/scripts"

Notes

  • If appDir is set, then baseUrl should be specified as relative to the appDir.
close
mainConfigFile
Path of requirejs configuration file to be passed for the optimization build.
notes

Usage

  • Example - mainConfigFile mainConfigFile: "../app/public/scripts/config/requireConfig.js"

Notes

  • The path to mainConfigFile will be relative to the build script file
  • The path specified, must include the ".js" extension
close
paths
Set paths for modules
notes

Usage

  • Example - specifying paths relative to baseUrl paths: { "jquery": "lib/jquery-2.1.0", "helloWorld": "modules/helloWorld" "weatherChennai": "empty:", // a http call to get weather data "jqueryCDN": "empty:" // a CDN resource }

Notes

  • Used to set module paths, if mainConfigFile is not provided.
  • "empty:" maps module names that are CDN resources on a CDN or http calls
  • During optimization "empty:" ensures that file is skipped because it has no dependencies.
close

dir
The directory to save the optimized build file(s)
notes

Usage

  • Example: specifying the build output directory dir: "../build/"

Notes

  • If dir is not specified, then path will default to be a directory called "build" as a sibling to the build file.
close
keepBuildDir
Do not delete the existing contents of directory specified by dir config option
notes

Usage

  • Example: specifying keepBuildDir keepBuildDir: true // do not delete the existing contents in the directory

Notes

  • Default value is false
  • The directory specified by dir config option will be deleted before the build starts again. This can avoided by setting keepBuildDir to true
close
shim
Specify the shim config option for ordering and dependencies
notes

Usage

  • Example: specifying shim config shim: { "maths": { exports: "simpleMATH", init: function () { simpleMATH = this.maths; } }, }

Notes

  • If shim config is used in the app during runtime, duplicate the config options under shim
  • Using mainConfigFile is a better way to pass this information though, so that it is only listed in one place.
close
wrapShim
Wraps shimmed dependencies within define(), which inturn is wrapped by a anonymous intermediate function
notes

Usage

  • Example: specifying wrapShim wrapShim: true

Notes

  • Default value is false
close
locale
Inlines i18n resources will in the build
notes

Usage

  • Example: specifying locale locale: "ta-in"

Notes

  • Only one locale can be inlined for a build.
  • Root bundles referenced by a build layer will be included in a build layer regardless of locale being set
close
optimize
optimization method to be used for all the JS files in the build output directory.
notes

Usage

  • Example: specifying optimization method optimize: "uglify"

Notes

  • Optimization Methods
    • "uglify": (default) uses UglifyJS to minify the code.
    • "uglify2": in version 2.1.2+. Uses UglifyJS2.
    • "closure": uses Google's Closure Compiler in simple optimization. ( Only available, when run in Java Env )
    • "none": no minification will be done.
close
skipDirOptimize
If true, r.js will optimize only those scripts modules, specified as part of the modules config option
notes

Usage

  • Example: specifying optimization method modules: [ { name: "namedModule" } ], optimize: "uglify2", skipDirOptimize : true

Notes

  • skipDirOptimize, applies only if dir config option is specified. In other words, it works only when optimizing the app as a build directory.
  • Doesn't have any impact to the output build, if optimize config option is set to none
  • skipDirOptimize, dir, optimize config option(s) go together.
close
generateSourceMaps
generate source map for the minified code
notes

Usage

  • Example: enable generateSourceMaps optimize: "uglify2", preserveLicenseComments: false, generateSourceMaps: true

Notes

  • The source files will show up in a browser developer tool that supports source maps as ".js.src" files.
close
normalizeDirDefines
specify if normalization should occur only with bundled modules or for entire directory build.
possible values: "all", "skip"
notes

Usage

  • Example: normalizeDirDefines optimize: "uglify2", skipDirOptimize : false, normalizeDirDefines: "skip"

Notes

  • r.js offers an excellent build hook "onBuildRead" which is invoked for each module, the return value of this hook will be used for the built file prior to minification. For performance reasons r.js will only invoke this only on bundled modules by default. Setting normalizeDirDefines to "all" means these modifications will be run on all files, not just your bundled modules.
    Ref: http://merrickchristensen.com/articles/build-angular-with-requirejs.html
  • Normalization is not done if optimize is set to "none". i.e. normalizeDirDefines will be automatically set to "skip"
  • normalizeDirDefines is automatically set to "all", if optimize is not "none" and skipDirOptimize is false.
close

optimizeCss
Enable or disable CSS optimizations
notes

Usage

  • Example - optimizeCss config option optimizeCss: "standard.keepLines.keepWhitespace"

Notes

  • "none": skip CSS optimizations.
  • "standard": @import inlining and removal of comments, unnecessary whitespace and line returns.
    Removing line returns may have problems in IE, depending on the type of CSS.
  • "standard.keepLines": like "standard" but keeps line returns.
  • "standard.keepComments": keeps the file comments, but removes line returns.
  • "standard.keepComments.keepLines": keeps the file comments and line returns.
  • "standard.keepWhitespace": like "standard" but keeps unnecessary whitespace.
close
cssImportIgnore
If optimizeCss is in use, a list of files to ignore for the @import inlining.
notes

Usage

  • Example - don't ignore any css @import cssImportIgnore: null
  • Example - ignore the list of css @import cssImportIgnore: "grid.css,typography.css"

Notes

  • The value of this option should be a string of comma separated CSS file names to ignore (like 'a.css,b.css'. The file names should match whatever strings are used in the @import calls.
close
cssIn
Optimize a single CSS file
notes

Usage

  • Example - optimize a single css file cssIn: "path/to/main.css" , out: "path/to/css-optimized.css"

Notes

  • When using cssIn config option, use out config option to specify the output path and file name for the css
close
out
Specify the output file path and file name
notes

Usage

  • Example - specifying a single js file optimization name: "path/to/scriptModule" , out: "path/to/optimizedScriptModule.js"
  • Example - specify a single css file optimization cssIn: "path/to/main.css" , out: "path/to/optimizedFile.css"

Notes

  • out requires the file extension for the optimized file.
close

skipModuleInsertion
If skipModuleInsertion is false, then files that do not use define() to define modules will get a define() placeholder inserted for them.
notes

Usage

  • Example - skipModuleInsertion skipModuleInsertion: false,

Notes
